# 个人简历项目经历 - C++ AI 应用服务平台 / 智能 LeetCode 刷题助手

**核心技术栈**：C++11/14, HTTP/TCP, Muduo网络库, Epoll, 策略模式/工厂模式, RabbitMQ, MySQL (连接池), 本地/云端大模型接入, RAG, MCP工具协议, ASR/TTS

**项目背景**：
为了解决市面上缺乏完整的 C++ 原生 AI 应用工程化落地框架的问题，基于自研的 C++ HTTP 框架打造了一个高性能、可扩展的 AI 应用服务底座。该平台成功孵化了“智能 LeetCode 刷题助手”，不仅提供刷题进度管理与艾宾浩斯复盘功能，还利用 LLM 为用户提供代码逻辑分析与判题服务。

---

### 📌 版本一：投递【C++ 后端开发 / AI大模型开发】综合通用版（推荐版，兼顾双边）
*此版本将底层架构与AI工程化深度结合，既体现了后端架构的扎实基础，又凸显了AI场景工程落地的能力，适合同时投递两个方向的岗位。*

**项目职责与核心贡献**：
- **AI基座与高性能网络架构设计**：基于 **Muduo网络库 (Reactor模型 + 非阻塞 I/O + Epoll)** 构建高并发基础服务端，独立实现 HTTP 协议解析层 (HttpContext) 及动态路由分发 (Dispatcher)。通过封装 **MySQL 连接池**将海量题库和刷题记录的查询控制在毫秒级，实现底层网络收发与上层业务处理的分离解耦。
- **多模型兼容引擎与轻量化 MCP 落地**：依托**策略模式与基于注册的工厂模式**统一 `AIStrategy` 模型接口，实现一键平滑切换云端大模型（阿里云百炼、豆包等）及本地 LLaMA 推理；设计配置化的工具注册中心（`AIToolRegistry`），利用 Prompt 协议实现了模型的“意图识别 -> 工具调用 -> 二次融合”**两段式推理**，赋予 LLM 调用外部工具（如判题机、数据库查询）的能力。
- **高并发日志削峰与异步业务保障**：针对用户与大模型交互产生的大规模、长耗时 AI 会话日志落库问题，引入 **RabbitMQ** 消息队列实现数据的异步写入与读写分离，确保主线程网络处理免受阻塞，有效保护数据库并实现流量削峰填谷。
- **RAG（检索增强）与多模态链路扩展**：主导设计从文本解析、分块、嵌入向量化（Embedding）到高维相似度检索（兼容 Faiss/Milvus）的 RAG 核心数据流，有效利用知识库补全大模型回答引用。此外，系统集成百度语音服务 (ASR/TTS) 与本地 ONNX 图像识别模块，丰富了跨模态应用能力。
- **智能复习算法与隔离级会话管理**：底层整合用户“艾宾浩斯记忆曲线”算法完成学习状态智能推算；重构单用户单会话的限制，采用细粒度 `unordered_map<userId, map<sessionId, AIHelper>>` 精准管理并发上下文，并最终利用 Docker v1/v2 镜像完成 MySQL、MQ 及 C++ 后端的全套容器化统一部署。

### 📌 版本二：偏向【AI 大模型开发 / AI 工程化】方向
*突出你在 LLM 落地、RAG 检索增强技术、以及模型平台工具化能力。*

**项目职责与核心贡献**：
- **多模型统一接入引擎设计**：基于**策略模式（Strategy）与注册式工厂模式（Factory）**，实现统一的 `AIStrategy`。一键解耦并支持阿里云百炼、豆包、通义千问等主流大模型，且同时预留接入本地 LLaMA/llama.cpp 部署。
- **RAG（检索增强生成）全链路落地**：开发了完整的 RAG 数据流，涵盖文档解析、分块（Chunking）、词嵌入向量化（Embedding）、以及基于高维向量的相似度检索（兼容 Faiss/Milvus），最终整合 Prompt 实现带知识引用的高质量回答。
- **轻量级 MCP（Model Context Protocol）能力落地**：设计了配置化的工具注册中心（`AIToolRegistry`），利用 Prompt 协议规范实现了大模型的**两段式推理**（意图识别/模型判断 -> 工具调用执行 -> 二次融合回答），赋予模型调用外部工具的能力（如计算器、数据库查询等）。
- **多会话上下文管理机制**：重构系统的会话隔离方式，从传统的单用户会话升级为支持单用户多会话隔离，采用细粒度的 `unordered_map<userId, map<sessionId, AIHelper>>` 精准管理大模型上下文。
- **多模态链路集成**：对接语音链路（ASR/TTS）与图像识别服务（依托 ONNXRuntime 本地推理框架），支持百度 TTS 参数化调整语速音色，实现模型文本处理能力与语音视觉能力的结合。

### 📌 版本三：偏向【C++ 后端开发 / 服务端架构】方向
*突出你在高性能服务器、网络并发、数据库、消息队列解耦及服务端架构设计等传统后端能力。*

**项目职责与核心贡献**：
- **高性能 HTTP 服务端基建**：基于 **Muduo 网络库（Reactor 模型 + 非阻塞 I/O + Epoll 线程池机制）**作为基础支撑，设计独立的高性能 HttpContext 层负责 Http 协议解析处理，实现服务端的高并发吞吐能力。
- **动态路由分发机制实现**：设计并实现 Router/Dispatcher 路由分发层，将 HTTP 请求 URL 映射到对应的各类业务 Handler 上，成功实现了网络通信和具体业务逻辑（如刷题记录查询、代码判题）的深度解耦。
- **消息队列异步削峰架构设计**：为了避免 AI 对话日志写入或判题等长耗时动作阻塞主线程、击垮数据库，引入 **RabbitMQ** 承载大文本交互日志数据，实现了后台异步消息消费与入库，保障核心流程削峰填谷的高可靠。
- **结构化存储与数据库优化**：搭建 **MySQL 连接池**预先维护多组长连接，规避频繁 TCP 握手开销。针对海量做题记录实现了单表毫秒级检索及自研的**轻量级自动表结构迁移（Auto-Migration）** 功能。
- **业务算法落地与全套容器部署**：在后端融入用户“艾宾浩斯记忆曲线”算法自动推算下一次复习时间。为保障环境一致性，将其 MySQL + RabbitMQ 以及 C++ 后端封装为 v1/v2 Docker 镜像，实现一键容器化交付运行。
